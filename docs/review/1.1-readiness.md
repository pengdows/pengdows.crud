# 1.1 Readiness Notes

## Fix / Improve
- SQL command text caching invalidation: `SqlContainer` caches `_cachedCommandText` but `Query` is mutable, so edits after first execution can reuse stale SQL. Add invalidation or safer API.
- Reinstate parameter limit enforcement: add a guard using `MaxParameterLimit` so oversized queries fail early with a clear message.
- DataSource connection settings: `DbDataSource` connections bypass `ApplyConnectionSettings`, so read-only connection tweaks and provider-specific settings may be skipped. Decide whether to apply or document.
- Release hygiene: keep vendor/benchmark artifacts out of git; ensure `.gitignore` and `tools/verify-novendor` are part of release checks.

## Potential Adds
- Metrics export hook or interval callback to avoid per-event handler overhead.
- Docs section clarifying DataSource vs Factory behavior and read-only semantics.
- Behavior note for `IsolationProfile.SafeNonBlockingReads` throwing on PostgreSQL (if intended).
